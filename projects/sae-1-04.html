SAE 1.04 — Se présenter sur Internet

Création d’un site web personnel responsive à partir d’un template Bootstrap, adapté à une présentation professionnelle et publié sur GitHub Pages.

Programmer Administrer Surveiller

Principe / Contexte

Cette SAE consistait à créer individuellement un site web personnel, en s’appuyant sur un template Bootstrap puis en le modifiant afin de répondre à un besoin : se présenter de manière professionnelle (accueil, CV en HTML/CSS, portfolio) et publier le site sur GitHub Pages. Le site devait être responsive (au moins trois tailles d’affichage), respecter les recommandations W3C, n’utiliser que des chemins relatifs et proposer des liens fonctionnels (GitHub, LinkedIn, documents, etc.).

Objectifs principaux

  • Créer un site responsive (petit / moyen / grand écran) avec une mise en page cohérente.
  • Structurer le site (menu, pages, sections, footer, liens) et respecter les bonnes pratiques HTML/CSS.
  • Réaliser un CV en HTML/CSS (pas uniquement un PDF à télécharger).
  • Construire un portfolio basé sur les compétences PN R&T : Administrer, Connecter, Programmer, Sécuriser, Surveiller.
  • Publier sur GitHub Pages et vérifier l’accessibilité du site via l’URL publique.

Thèmes abordés

  • HTML : structure, sémantique, navigation, liens relatifs.
  • CSS : mise en forme, cohérence visuelle, responsive design.
  • Bootstrap : grille, composants, adaptation d’un template existant.
  • Publication : hébergement GitHub Pages, gestion des fichiers et de l’arborescence.
  • Qualité : liens fonctionnels, cohérence globale, respect des consignes (W3C, chemins relatifs).

Outils utilisés

  • HTML / CSS
  • Bootstrap
  • Éditeur de code
  • GitHub + GitHub Pages

Organisation et déroulement

  • Travail individuel à partir d’un template Bootstrap.
  • 2h encadrées : choix/validation du template, questions, premières réalisations.
  • 14h en autonomie : adaptation du template, création des pages et contenus.
  • 2h encadrées : évaluation écrite HTML/CSS, échanges et corrections sur le site.
  • Publication du site sur GitHub Pages et envoi du lien avant la date limite.

Difficultés rencontrées

  • Cohérence graphique : harmonisation des couleurs, des typographies et des images.
  • Gestion des chemins relatifs : organisation des fichiers et liens internes.
  • Template Bootstrap : suppression/ajustement d’éléments inutiles et correction d’incohérences héritées du template.
  • Qualité globale : quelques liens non fonctionnels et incohérences résiduelles, liées au fait que c’était l’une de mes premières SAE.
