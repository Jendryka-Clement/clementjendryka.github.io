SAE 2.04 — Projet Pépinière : Intégration & applicatif

Mise en situation “entreprise” : intégrer plusieurs briques techniques (web, base de données, capteurs/équipements, réseau et services) dans un délai court.

Administrer Connecter Programmer

Principe / Contexte

Cette SAE prolonge le projet “pépinière” avec une partie intégration / applicatif. Le but était de faire fonctionner ensemble plusieurs composants (infrastructure, services, page web, base de données de mesures, équipements type capteur/carte Arduino et caméra), en assurant la cohérence réseau et la configuration associée.

Objectifs principaux

  • Intégrer les différents composants d’un système (réseau, services, applicatif, données et équipements).
  • Assurer la communication entre les briques (cohérence réseau, routage, configurations).
  • Mettre en place une partie applicative (page web) connectée à une base de données (ex. mesures de température).
  • Documenter l’intégration, les configurations et les tests réalisés.

Thèmes abordés

  • Intégration système : web + base de données + équipements + réseau.
  • Communication et cohérence réseau : routage, configurations, validation.
  • Services et environnement serveur (ex. Windows Server si utilisé dans le projet).
  • Organisation projet “entreprise” : coordination, délais, livrables.

Outils utilisés

  • Environnement réseau / serveurs / services (selon la maquette du projet).
  • Page web + base de données (mesures type température).
  • Équipements du projet (ex. carte Arduino/capteur, caméra) et configurations associées.
  • Outils de documentation : schémas, comptes-rendus, journal de bord.

Organisation et déroulement

  • Répartition des tâches : infra/services, intégration réseau, applicatif, base de données, équipements, tests.
  • Mise en place progressive et validations par itérations (tests + corrections).
  • Documentation continue pour garder une trace des configurations et des choix.
  • Finalisation : rendu, rapport, journal de bord et soutenance.

Difficultés rencontrées

  • Manque de temps : intégration multi-briques très dense dans une durée courte.
  • Incident majeur : après 3 jours de travail, l’environnement a “planté” et il a fallu recommencer depuis le début.
  • Difficulté d’intégration : relier base de données (température), carte Arduino/capteur, caméra, page web, routage et configurations, services (ex. Windows Server).
  • Contrainte d’espace : travail “les uns sur les autres”, moins de confort et de fluidité.

Ce que la SAE m’a appris

  • Travailler en mode “projet d’entreprise” : coordination, délais, livrables, contraintes.
  • Intégrer plusieurs briques techniques dans un système complet et cohérent.
  • Gérer des imprévus et prioriser pour livrer un résultat exploitable.
