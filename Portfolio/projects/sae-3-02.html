SAE 3.02 — Développer des applications communicantes

Développement d’une application client/serveur en Python 3 permettant de rechercher des mots dans plusieurs formats (texte, HTML, PDF, Excel) et d’afficher les résultats dans une interface Tkinter, avec une documentation de projet.

Programmer Administrer Surveiller

Principe / Contexte

Le but de la SAE 3.02 était de concevoir une application en Python 3 capable de communiquer en mode client/serveur et de réaliser une recherche de mots dans plusieurs types de documents (TXT, HTML, PDF, Excel). Les résultats devaient être affichés dans une interface graphique (Tkinter), avec un code structuré et une documentation.

Objectifs principaux

  • Mettre en place une architecture client/serveur (communication réseau).
  • Gérer plusieurs formats de fichiers (lecture/parsing) de manière fiable.
  • Afficher des résultats exploitables (lignes, occurrences, contexte) dans une interface Tkinter.
  • Documenter le projet (README, dépendances, explications d’usage).

Thèmes abordés

  • Réseau : sockets TCP, échanges client/serveur.
  • Programmation : structuration, gestion d’erreurs, modularité.
  • Traitement de données : parsing HTML, extraction PDF, lecture Excel.
  • Interface : Tkinter (ergonomie et lisibilité des résultats).

Outils utilisés

  • Python 3
  • Tkinter
  • Bibliothèques de parsing/lecture (selon ton implémentation : HTML / PDF / Excel)
  • Git (dépôt, suivi du projet) + documentation (README, requirements)

Organisation et déroulement

  • Conception : définition des formats supportés et du format de sortie.
  • Développement itératif : implémentation par format (TXT → HTML → PDF → Excel).
  • Tests : validation avec des fichiers de référence et cas limites.
  • Industrialisation : nettoyage du code, structuration, documentation.

Difficultés rencontrées

  • Extraction PDF : texte non linéaire, retours à la ligne imprévisibles, encodages.
  • Recherche : gestion des occurrences, mots “collés”, casse, et limites possibles de la logique de recherche.
  • Affichage : rendre les résultats lisibles dans Tkinter (liste, surlignage, contexte).

Auto-évaluation

Niveau : C. Certaines consignes et explications n’étaient pas suffisamment claires, ce qui a demandé du temps pour comprendre ce qui était réellement attendu et comment structurer la solution. Au-delà du développement, j’ai trouvé que l’utilité “métier” de la SAE restait limitée : le projet permet de coder, mais l’exercice aurait eu plus de sens s’il avait été intégré à une SAE plus importante, avec un contexte plus concret et des objectifs plus directement exploitables.
