SAE 5.03 — Déployer et gérer les services ROM

Projet : déploiement d’Asterisk sur Alpine, intégration Web (sip.js / WebRTC) et construction d’un portail de visio-conférence (voix/vidéo/messagerie), avec validations et audit.

Administrer Connecter Programmer Surveiller Sécuriser
Type : Travail en binôme
Année : 2025–2026
Thématique : VoIP / WebRTC / Services ROM
Livrables : Rapport technique + démonstrations (vidéos) + soutenance
Objectif global : déployer un serveur Asterisk (Alpine) et intégrer ses services dans une interface web, jusqu’à un portail de visio-conférence (voix, vidéo, messagerie).

Principe / Contexte
Le projet consistait à mettre en place une solution de téléphonie et de communication temps réel “comme en entreprise”, en partant d’un serveur Asterisk déployé sur une machine virtuelle Alpine. Ensuite, l’objectif était d’intégrer la voix (puis la vidéo et la messagerie) via une interface web, jusqu’à obtenir un portail de visio-conférence exploitable et analysable (tests, captures, audit).
Objectifs principaux
  • Déployer et valider un serveur Asterisk sur une VM Alpine.
  • Connecter des terminaux SIP (téléphones IP Cisco/Fanvil) et valider les appels voix.
  • Mettre en place l’administration HTTP d’Asterisk et développer une interface web simple.
  • Déployer un serveur web sur Alpine et intégrer une page Web de communication voix via sip.js.
  • Étendre l’interface : vidéo + messagerie, puis portail de visio-conférence.
  • Réaliser une démarche de tests et un audit sur les échanges portail web / Asterisk.
Thèmes abordés
  • VoIP / SIP : enregistrement, comptes, plan de numérotation, boîtes vocales.
  • Déploiement système : VM Alpine, services, configuration et validation.
  • WebRTC côté web : communication temps réel dans le navigateur.
  • Développement web : intégration sip.js, interface, portail.
  • Analyse et contrôle : captures Wireshark, compréhension des flux, audit sécurité.
Outils utilisés
  • Alpine Linux (machines virtuelles dédiées).
  • Asterisk (serveur de téléphonie), terminaux IP Cisco et Fanvil.
  • Serveur web (Apache ou Nginx) + pages web d’intégration.
  • sip.js + WebRTC (voix/vidéo dans le navigateur).
  • Wireshark (validation, analyse de protocoles, démonstration des échanges).
  • TFTP (pour l’enregistrement/configuration de certains terminaux).
Organisation et déroulement
  • Projet découpé en étapes successives, chacune validée avant de poursuivre.
  • Déploiement Asterisk (Alpine) → intégration des téléphones → interface web d’administration.
  • Déploiement serveur web → intégration sip.js (voix) → ajout vidéo/messagerie.
  • Réalisation d’un portail de visio-conférence, puis validations et audit.
Difficultés rencontrées
  • Blocages techniques lors de l’intégration complète (Asterisk + terminaux + web + WebRTC).
  • Difficulté à stabiliser une solution “bout-en-bout” : quand une brique ne fonctionne pas, tout le scénario est bloqué.
  • Temps consommé par le debug réseau/protocoles et par les réglages (enregistrement SIP, TFTP, flux WebRTC).

Ce que la SAE m’a appris
  • Déployer un service VoIP de manière propre (OS minimal + service + validation).
  • Comprendre les dépendances “multi-briques” (SIP, web, WebRTC, terminaux, réseau).
  • Structurer un projet par étapes et documenter les validations (tests, captures, démonstrations).
