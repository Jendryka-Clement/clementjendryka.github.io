SAE 5.01 — Concevoir, réaliser et présenter une solution technique

Projet : salles de TP virtuelles à l’aide de conteneurs (plateforme centralisée, accès web, noVNC, Windows Server intégré).

Principe / Contexte
L’objectif de cette SAE était de concevoir une plateforme unique permettant d’accéder à des environnements de TP (programmation, réseaux, cybersécurité) sous forme de conteneurs. L’accès devait se faire simplement depuis un navigateur, via une interface web, avec une gestion des droits (utilisateur / administrateur). La solution devait également inclure un environnement Windows Server intégré à l’écosystème pour des usages pédagogiques.
Objectifs principaux
  • Déployer une plateforme complète sur une seule VM (déploiement reproductible).
  • Créer une interface web centralisée : connexion, accès, lancement des environnements.
  • Fournir plusieurs environnements Linux adaptés aux TP (Prog / Réseaux / Cyber).
  • Permettre un accès “graphique” dans le navigateur via noVNC.
  • Intégrer un Windows Server dans la solution et garantir un accès fonctionnel.
  • Mettre en place des bases de sécurité (gestion des comptes, séparation des rôles, isolation).
Thèmes abordés
  • Conteneurisation et déploiement : organisation des services et des images.
  • Développement d’un portail web : authentification, gestion de sessions, droits.
  • Accès distant à un bureau graphique : noVNC et environnements Linux.
  • Intégration Windows Server dans une infrastructure Linux (virtualisation).
  • Sécurité et exploitation : isolation, logique de rôles, stabilité et maintenance.
Outils utilisés
  • Docker et Docker Compose (déploiement et orchestration de la plateforme).
  • Flask (portail web : connexion, rôles, navigation vers les environnements).
  • noVNC (accès navigateur aux environnements graphiques).
  • Dockur / QEMU (intégration de Windows Server).
  • Outils de diagnostic (logs, vérifications réseau, tests de stabilité).
Organisation et déroulement
  • Définition de l’architecture et des services nécessaires (portail, environnements, accès).
  • Développement du portail web et mise en place des rôles (admin / user).
  • Création des environnements Linux et intégration de l’accès noVNC.
  • Ajout et mise au point de l’environnement Windows Server.
  • Tests complets, corrections, stabilisation, puis rédaction et démonstration.
Difficultés rencontrées
  • Contraintes d’environnement (réseau/proxy) qui peuvent bloquer téléchargements et builds.
  • Windows Server : contraintes de ressources (disque/CPU), temps de démarrage et réglages de virtualisation.
  • Stabilité de certains services : conteneurs qui redémarrent, ajustements de configuration nécessaires.
  • Arbitrages techniques : faire simple et robuste pour une utilisation en TP, sans complexité inutile.

Ce que la SAE m’a appris
  • Concevoir une architecture cohérente et la rendre déployable simplement (vision “solution”).
  • Faire le lien entre applicatif (portail web) et infrastructure (conteneurs) de manière propre.
  • Gérer des contraintes techniques réelles : ressources, stabilité, réseau, arbitrages.
  • Industrialiser : tester, corriger, documenter, puis démontrer une solution fonctionnelle.
