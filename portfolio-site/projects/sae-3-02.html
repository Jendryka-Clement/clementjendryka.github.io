Principe / Contexte

Objectif : développer une application en Python 3 permettant de rechercher un ou plusieurs mots dans différents types de documents (texte, HTML, PDF, Excel) et d’afficher les résultats dans une interface graphique. Le projet doit respecter des bonnes pratiques de développement (structure, lisibilité, documentation).

Objectifs principaux

  • Mettre en place une architecture client/serveur (communication réseau).
  • Gérer plusieurs formats de fichiers (parsing/lecture) de manière fiable.
  • Afficher des résultats exploitables (lignes, occurrences, contexte) dans une GUI Tkinter.
  • Documenter le projet (README en anglais, requirements.txt).

Thèmes abordés

  • Réseau : sockets TCP, échanges client/serveur.
  • Programmation : structuration, exceptions, modularité.
  • Traitement de données : parsing HTML, extraction PDF, lecture Excel.
  • Interface : Tkinter, ergonomie minimale.

Outils utilisés

  • Python 3
  • Tkinter
  • BeautifulSoup (HTML)
  • PyPDF2 (PDF)
  • openpyxl / pandas (Excel)
  • GitLab (suivi + dépôt), README, requirements.txt

Organisation et déroulement

  • Conception : définition des formats supportés et du format de sortie.
  • Développement itératif : implémentation par format (TXT → HTML → PDF → Excel).
  • Tests : validation avec des fichiers de référence et cas limites.
  • Industrialisation : nettoyage du code, PEP8, doc utilisateur (README).

Difficultés rencontrées

  • Extraction PDF : texte non linéaire, retours à la ligne imprévisibles, encodages.
  • Recherche : gestion des occurrences, mots collés, casse, éventuelles regex.
  • Affichage : rendre les résultats lisibles dans Tkinter (liste, surlignage, contexte).
